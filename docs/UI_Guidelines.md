# UI Guidelines

## Product Feel

VCC-OS should feel calm, premium, fast, and operational. It should help the user scan their situation quickly and choose the next action.

## Style Direction

- Dark command-center baseline.
- Clear cards and panels.
- Large important numbers.
- Minimal clutter.
- Strong status colors:
  - Green for good/safe/complete.
  - Orange/yellow for warning.
  - Red for critical.
  - Blue for informational actions.

## Layout Rules

- Dashboard is a command center, not an editing surface.
- Dedicated pages handle detailed module work.
- Preserve mobile-first readability.
- Avoid redesigning the whole layout during feature sprints.
- Keep visual hierarchy consistent across modules.

## Component Rules

- Use existing UI primitives where available.
- Prefer semantic buttons, links, tables, and forms.
- Keep touch targets usable.
- Avoid hiding critical actions behind unclear icons.
- Use consistent empty states and loading states.

## Accessibility

- Interactive elements must be keyboard reachable.
- Use visible focus states.
- Do not rely on color alone for critical status.
- Tables should preserve headers and readable cell labels.
