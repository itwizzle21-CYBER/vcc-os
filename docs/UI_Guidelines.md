# UI Guidelines

## Product Feel

VCC-OS should feel like a calm, dense, premium command center. It should prioritize scanning, comparison, and repeated use.

## Layout Rules

- Do not redesign the whole layout during focused sprints.
- Dashboard is read-only.
- Editing belongs in dedicated pages.
- Preserve desktop sidebar and mobile bottom navigation.
- Keep touch targets usable.
- Avoid nested card-heavy layouts.

## Visual Rules

- Use the current custom CSS skin in `src/styles/vccSkin.css`.
- Keep typography compact and readable.
- Use status color consistently:
  - Red: critical
  - Amber/yellow: warning
  - Green: stable/good
  - Accent: active/focus
- Avoid text overlap.
- Keep spreadsheet grids horizontally usable.

## Accessibility Rules

- Use real buttons for actions.
- Keep focus states visible.
- Do not rely on color alone for meaning.
- Labels and button text must fit on mobile and desktop.
