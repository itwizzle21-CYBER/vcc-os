## How To Use During VCC Sprints

Use skills intentionally by sprint phase:

1. Product and UI planning: use `design-systems`, `design-critique`, `emil-design-eng`, `data-viz`, and `responsive-layout`.
2. Component implementation: use `frontend-code-review`, `component-states`, `a11y-pass`, `micro-motion`, and `animation-vocabulary`.
3. Motion review: use `review-animations` when animation or Framer Motion behavior is introduced.
4. Financial dashboard work: use `data-viz`, `database-testing`, `risk-based-testing`, and existing VCC financial standards.
5. Browser QA: use `playwright`, `playwright-interactive`, `playwright-automation`, `visual-testing`, and `screenshot`.
6. Production readiness: use `performance-testing`, `release-readiness`, `gh-fix-ci`, `gh-address-comments`, and `vercel-deploy`.
7. Security and architecture: use `security-best-practices`, `security-threat-model`, `security-ownership-map`, and `frontend-code-review`.
8. Release and PR flow: use `yeet` only when explicitly asking Codex to stage, commit, push, and open a PR in one flow.

Restart Codex to pick up newly installed skills.
